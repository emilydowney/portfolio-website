Overview

Pokédex is a JavaScript focused web application, able to load and view data from an external API. Users can search through and click on Pokémon to reveal more information about each one.

Purpose

Pokédex is a project built as part of my web development coursework through CareerFoundry. The goal of the project was to build a functioning, JavaScript web app to display in my portfolio. My goal was to feature clean code and design.

Tools

  • HTML/CSS
  • JavaScript/jQuery
  • Bootstrap
  • ESLint
  • Autoprefixer CSS

The Process

Step 1: Using an External API

The earlist stages of this project involved choosing an external API to draw and display data from. The Pokémon API was chosen because it's a public API with extensive documentation. I first retrieved the Pokémon data using the fetch() function before using a forEach loop to display each of the Pokémon names. Asynchronous behavior was implemented for a cleaner app and improved usability. I chose to pull the main details of each Pokémon from the API, feeling the full list of information was beyond the scope of this project.

Step 2: UI Elements and jQuery

This application implements a UI pattern called a modal to overlay information of each Pokémon upon selection. The coding for this part of the project was challenging for me, but the use of the jQuery library helped streamline and simplify the process. Functions are used to open and close the modal in response to mouse clicks. I also took time in this part of the project to organize my code with spacing and notes to improve the code's readbility. I've learned it's important to implement this step early for easier navigation as the project expands.

Step 3: Bootstrap Implementation

Responsiveness is a vital piece of any web application due to the variety of screen sizes and devices users have access to. The addition of the Bootstrap framework allowed me to create a clean design while easily adapting to window size changes. I used Bootstrap primarily to create the navbar and modals, combining Bootstrap's prebuilt components with my own designs.

Step 4: Final Design and Accessibility

In the late stages of this project, I finalized my design decisions. I chose a color scheme that complimented the Pokémon brand without being overly messy. In addition, I formatted the information modals with left-aligned text and clean, centered titles. I also checked the colors and HTML attributes to insure my app fit with accessibility standards. To finish the project, I used ESLint and W3C Markup to check my code for bugs and errors.

Conclusion

Overall, I am happy with the result of my Pokédex project. I met the assignment goals concerning functionality and user experience. Looking back, I would add a few changes to the project to improve it’s design. I think the app would benefit from more sorting options, such as looking through the Pokémon alphabetically or by type. I would also add some sort of design to the Pokémon types to add some distinction to the text.
